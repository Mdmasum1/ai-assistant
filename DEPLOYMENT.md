# Deployment Guide for AI Voice Assistant Landing Page

This guide explains how to deploy your AI Voice Assistant landing page to Render.com and other platforms.

## Render.com Deployment

### Prerequisites
- GitHub account
- Render.com account (free tier available)
- Your project code pushed to a GitHub repository

### Step-by-Step Deployment

1. **Prepare Your Repository**
   - Ensure all files are committed to your GitHub repository
   - Verify that `requirements.txt`, `render.yaml`, and `Procfile` are in the root directory

2. **Connect to Render**
   - Go to [render.com](https://render.com) and sign in
   - Click "New +" and select "Web Service"
   - Connect your GitHub repository

3. **Configure the Service**
   - **Name**: `ai-voice-assistant` (or your preferred name)
   - **Environment**: `Python`
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `python src/main.py`
   - **Plan**: Free (or upgrade as needed)

4. **Environment Variables**
   - Set `FLASK_ENV` to `production`
   - Render will automatically generate a `SECRET_KEY`

5. **Deploy**
   - Click "Create Web Service"
   - Render will automatically build and deploy your application
   - Your app will be available at `https://your-app-name.onrender.com`

### Render.yaml Configuration

The included `render.yaml` file automatically configures:
- Python environment
- Build and start commands
- Environment variables
- Health check endpoint

## Alternative Deployment Options

### Heroku
1. Install Heroku CLI
2. Create a new Heroku app: `heroku create your-app-name`
3. Push to Heroku: `git push heroku main`
4. Set environment variables: `heroku config:set FLASK_ENV=production`

### Vercel
1. Install Vercel CLI: `npm i -g vercel`
2. Run `vercel` in your project directory
3. Follow the prompts to deploy

### Railway
1. Connect your GitHub repository to Railway
2. Railway will automatically detect and deploy your Flask app
3. Configure environment variables in the Railway dashboard

## Custom Domain Setup

### For Render.com
1. Go to your service dashboard
2. Click "Settings" → "Custom Domains"
3. Add your domain and configure DNS records
4. Enable SSL (automatic with Render)

### DNS Configuration
Point your domain to your deployment platform:
- **Render**: Use the provided CNAME or A record
- **Heroku**: CNAME to `your-app.herokuapp.com`
- **Vercel**: Follow Vercel's domain configuration guide

## Environment Variables

### Required Variables
- `FLASK_ENV`: Set to `production` for live deployment
- `SECRET_KEY`: Automatically generated by Render, or set manually
- `PORT`: Automatically set by most platforms

### Optional Variables
- `DATABASE_URL`: If using a production database
- `REDIS_URL`: If using Redis for caching

## Performance Optimization

### For Production
1. **Enable Gzip Compression**
2. **Use a CDN** for static assets
3. **Implement Caching** headers
4. **Monitor Performance** with tools like New Relic or DataDog

### Database Considerations
- The current setup uses SQLite (suitable for demos)
- For production, consider PostgreSQL or MySQL
- Update `SQLALCHEMY_DATABASE_URI` for production database

## SSL/HTTPS
- Render.com provides automatic SSL certificates
- Ensure all external links use HTTPS
- Update any hardcoded HTTP URLs to HTTPS

## Monitoring and Logs

### Render.com
- View logs in the Render dashboard
- Set up log drains for external monitoring
- Configure health checks and alerts

### Error Tracking
Consider integrating:
- Sentry for error tracking
- LogRocket for user session recording
- Google Analytics for traffic monitoring

## Scaling Considerations

### Horizontal Scaling
- Render supports auto-scaling based on traffic
- Configure scaling rules in the dashboard
- Monitor resource usage and adjust as needed

### Database Scaling
- Move from SQLite to PostgreSQL for better performance
- Consider read replicas for high-traffic applications
- Implement connection pooling

## Security Best Practices

1. **Environment Variables**: Never commit secrets to version control
2. **HTTPS Only**: Ensure all traffic uses HTTPS
3. **Security Headers**: Implement proper security headers
4. **Rate Limiting**: Add rate limiting for API endpoints
5. **Input Validation**: Validate all user inputs

## Troubleshooting

### Common Issues
1. **Build Failures**: Check `requirements.txt` for correct dependencies
2. **Port Issues**: Ensure app listens on `0.0.0.0` and uses `PORT` env var
3. **Static Files**: Verify static file paths are correct
4. **Database Errors**: Check database connection and migrations

### Debug Mode
- Never enable debug mode in production
- Use logging instead of print statements
- Implement proper error handling

## Backup and Recovery

### Database Backups
- Set up automated database backups
- Test restore procedures regularly
- Keep backups in multiple locations

### Code Backups
- Use version control (Git) for code
- Tag releases for easy rollback
- Maintain staging and production branches

## Cost Optimization

### Free Tier Limits
- Render: 750 hours/month on free tier
- Heroku: 550-1000 hours/month on free tier
- Monitor usage to avoid unexpected charges

### Paid Plans
- Upgrade when you exceed free tier limits
- Consider reserved instances for predictable traffic
- Monitor costs and optimize resource usage
